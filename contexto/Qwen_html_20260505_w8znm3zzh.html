2. Manual de Usuario Completo

2.1 Introducción al Sistema

SenaKan es una plataforma web tipo Kanban diseñada para la coordinación interna del SENA. Permite visualizar, asignar y dar seguimiento a tareas académicas y administrativas mediante tableros interactivos.

2.2 Requisitos del Cliente

  • Navegador moderno: Chrome 90+, Firefox 88+, Edge 90+
  • Conexión a la red local del centro de formación
  • Resolución mínima: 1280x720 (optimizado para escritorio y tablets)

2.3 Acceso y Roles

Ingrese a http://localhost/senakan con su correo institucional y contraseña.

RolFunciones PrincipalesRestricciones
AdministradorGestión global de usuarios, roles y configuraciónSin restricciones
CoordinadorCrea proyectos/tableros, asigna tareas, ve reportesNo modifica configuración de sistema
InstructorActualiza estado de tareas propias, agrega comentariosNo elimina tareas de otros
AprendizVisualiza y actualiza tareas asignadasRequiere aprobación para mover fuera de columna

2.4 Funciones Principales

2.4.1 Tablero Kanban

  1. Acceda al menú Tableros y seleccione un proyecto.
  2. Las columnas por defecto son: Por Hacer, En Progreso, En Revisión, Completado.
  3. Arrastre una tarjeta entre columnas para actualizar su estado (RF-006).

2.4.2 Crear/Editar Tarea

  1. Haga clic en + Nueva Tarea.
  2. Complete: Título, Descripción, Asignado, Prioridad (Alta/Media/Baja), Fecha Límite.
  3. Guarde. La tarea aparecerá en "Por Hacer".

2.4.3 Comentarios y Notificaciones

Abra una tarjeta para ver el historial y agregar comentarios. Las notificaciones aparecen en la campana superior derecha al ser asignado o cuando se acerca la fecha límite (RF-008).

2.5 Preguntas Frecuentes y Solución de Problemas

  • ¿No puedo arrastrar tareas? Verifique que su rol tenga permisos de movimiento o contacte al Coordinador.
  • La fecha límite aparece en rojo: La tarea está vencida. Actualice el estado o ajuste la fecha si tiene permisos.
  • ¿Cómo recuperar contraseña? Use el enlace "¿Olvidó su contraseña?" en el login (requiere configuración SMTP previa).

3. Guía Rápida de Inicio (Quick Start)

Objetivo: Registrar y mover una tarea en el tablero Kanban en 5 minutos.

  1. Inicie sesión con sus credenciales en http://localhost/senakan/login.
  2. En el Dashboard, haga clic en un tablero disponible (ej. "PROYECTOS FORMATIVOS").
  3. Presione el botón verde + Nueva Tarea en la columna "Por Hacer".
  4. Complete el título, seleccione prioridad y asignado. Haga clic en Guardar.
  5. Arrastre la tarjeta a "En Progreso". El sistema actualizará el estado automáticamente y registrará el cambio en el historial.
⚠️ Nota: Solo los roles Coordinador e Instructor pueden crear tareas. Los Aprendices pueden mover tareas dentro de las columnas permitidas.

4. Manual Técnico del Software

4.1 Arquitectura de Software

SenaKan sigue el patrón Modelo-Vista-Controlador (MVC) implementado por Laravel 11:

  • Modelos (Eloquent ORM): User, Project, Column, Task, Comment, ActivityLog, Notification
  • Vistas (Blade + Bootstrap 5 + SortableJS): Renderizado en servidor con interacción AJAX para Drag & Drop.
  • Controladores: Rutas agrupadas por middleware auth y role:admin,coordinator. Validación con Form Requests.

4.2 Stack Tecnológico

CapaTecnologíaVersión
BackendPHP / Laravel≥8.1 / 11.x
Base de DatosMySQL / MariaDB8.0 / 10.4+
FrontendBlade, Bootstrap 5, SortableJSÚltimas estables
AutenticaciónLaravel Breeze / JWT (opcional)Session-based
ServidorApache 2.4 (XAMPP)Configuración local

4.3 Estructura de Base de Datos

Relaciones principales (ver kanban_sena.sql):

  • users → 1:N → tasks (assigned_to, created_by)
  • projects → 1:N → tasks & columns
  • tasks → 1:N → comments, activity_logs
  • Soft deletes implementados en tareas para evitar pérdida de datos.

4.4 Endpoints y Rutas Principales

GET /dashboard → Vista resumen por rol POST /tasks → Crear tarea (validación FormRequest) PATCH /tasks/{id}/status → Actualizar estado vía AJAX (RF-006) POST /tasks/{id}/comments → Agregar comentario (RF-007) GET /reports → Métricas por ficha/usuario (RF-010)

4.5 Seguridad y Mantenibilidad

  • Protección CSRF en todos los formularios.
  • Hashing de contraseñas con bcrypt.
  • Políticas de autorización (Gates/Policies) por rol.
  • Código documentado con PHPDoc, sigue PSR-12.

5. Manual de Instalación Paso a Paso

5.1 Requisitos Previos

  • Sistema: Windows 10/11 o Ubuntu 20.04+
  • XAMPP (Apache 2.4, MySQL 8.0+, PHP 8.1+)
  • Composer 2.x
  • Git (opcional, para control de versiones)

5.2 Procedimiento

  1. Instalar XAMPP: Descargue desde apachefriends.org. Instale con permisos de administrador.
  2. Iniciar servicios: Abra XAMPP Control Panel y active Apache y MySQL.
  3. Crear base de datos: Acceda a http://localhost/phpmyadmin → Nueva BD: kanban_sena → Cotejamiento: utf8mb4_unicode_ci.
  4. Desplegar código: Copie la carpeta del proyecto a C:\xampp\htdocs\senakan.
  5. Instalar dependencias:
    cd C:\xampp\htdocs\senakan
    composer install
  6. Configurar entorno: Renombre .env.example a .env y edite:
    DB_DATABASE=kanban_sena
    DB_USERNAME=root
    DB_PASSWORD=
  7. Generar clave y migrar:
    php artisan key:generate
    php artisan migrate
  8. Sembrar datos base:
    php artisan db:seed --class=RoleSeeder
  9. Verificar: Acceda a http://localhost/senakan. Credenciales por defecto: admin@sena.demo / sena1234.

5.3 Solución de Problemas Comunes

  • Error 500: Verifique permisos de carpetas storage y bootstrap/cache.
  • Puerto 80 ocupado: Detenga Skype/IIS o cambie el puerto en httpd.conf.
  • Error de migración: Asegure que MySQL esté corriendo y que .env tenga credenciales correctas.

6. Plan de Migración de Datos

6.1 Objetivo

Transferir tareas, fichas y asignaciones desde hojas de cálculo Excel/CSV a la base de datos MySQL de SenaKan, garantizando integridad y trazabilidad.

6.2 Mapeo de Datos

Origen (Excel)Destino (MySQL)Tipo de Transformación
Nombre Tareatasks.titleTrim + validación UTF-8
Descripcióntasks.descriptionHTML escape si contiene saltos
Estado (Texto)tasks.statusMapeo: "Pendiente"→pending, "En curso"→progress, "Listo"→done
Prioridadtasks.priorityhigh/medium/low
Fecha Límitetasks.due_dateFormato Y-m-d
Ficha/Proyectoprojects.codeUnicidad validada

6.3 Estrategia de Ejecución

  1. Limpieza de datos en Excel (eliminar duplicados, validar fechas).
  2. Exportar a CSV UTF-8.
  3. Ejecutar seeder personalizado: php artisan db:seed --class=MigrationFromExcelSeeder
  4. Validar conteo: SELECT COUNT(*) FROM tasks; debe coincidir con filas importadas.
  5. Registrar en activity_logs acción bulk_import.

6.4 Plan de Contingencia

Si la importación falla, se ejecuta php artisan migrate:rollback y se restaura backup previo. La migración se realiza en ventana de mantenimiento (viernes 18:00-20:00).

7. Plan de Respaldo y Restauración

7.1 Política de Backups

  • Frecuencia: Diario (automático) + Semanal completo.
  • Método: mysqldump comprimido + backup de archivos storage/ y .env.
  • Almacenamiento: Disco local externo + carpeta red del centro SENA.

7.2 Procedimiento de Backup

@echo off set DATE=%date:~6,4%%date:~3,2%%date:~0,2% "C:\xampp\mysql\bin\mysqldump.exe" -u root --databases kanban_sena > "C:\backups\senakan_%DATE%.sql" "C:\Program Files\7-Zip\7z.exe" a -tzip "C:\backups\senakan_%DATE%.zip" "C:\backups\senakan_%DATE%.sql"

7.3 Procedimiento de Restauración

  1. Detener Apache en XAMPP.
  2. Descomprimir backup.
  3. Ejecutar: mysql -u root kanban_sena < backup_YYYYMMDD.sql
  4. Reiniciar servicios y verificar integridad.

Prueba periódica: Mensualmente, restaurar en entorno de pruebas y validar login + tablero.

8. Procedimiento de Rollback y Contingencia

8.1 Escenarios de Falla

  • Error crítico post-actualización de código.
  • Corrupción de base de datos tras migración fallida.
  • Vulnerabilidad de seguridad detectada.

8.2 Pasos de Reversión

  1. Activar modo mantenimiento: php artisan down --message="Mantenimiento programado"
  2. Revertir código: git checkout <versión-anterior>
  3. Restaurar BD: Ejecutar backup del día anterior.
  4. Limpiar caché: php artisan cache:clear && php artisan config:clear
  5. Reactivar: php artisan up
  6. Notificar: Enviar comunicación a Coordinadores vía sistema y correo.

SLA de restauración: ≤ 45 minutos para rollback completo en entorno local.

9.3 Incidencias y Soluciones

  • Problema: SortableJS no sincronizaba estado con BD en Firefox.
    Solución: Implementar fetch API con CSRF token y validación en backend.
  • Problema: Lentitud en carga de reportes con +500 tareas.
    Solución: Optimizar consultas Eloquent con with() y paginación.

9.4 Resultados de Pruebas de Aceptación

Se ejecutaron 24 casos de prueba (CP-001 a CP-024). Tasa de éxito: 100%. Validado por Coordinador Académico y 3 instructores piloto.

9.5 Lecciones Aprendidas

  • La capacitación temprana reduce la curva de adopción en usuarios no técnicos.
  • Documentar el entorno local XAMPP evita incidencias de compatibilidad.
  • Los reportes PDF con DomPDF requieren optimización de memoria para grandes volúmenes.
